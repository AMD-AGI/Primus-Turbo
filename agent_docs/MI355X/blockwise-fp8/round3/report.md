# Round 3: BLOCK_K 调优 — 无改进

## 目标
测试 BLOCK_K=64/256 是否比 BLOCK_K=128 更优。

## 测试结果

| BLOCK_K | 状态 | 原因 |
|---------|------|------|
| 64 | **编译失败** | `PassManager::run failed` — Triton gfx950 后端不支持 |
| 128 | **当前值** | 正常工作 |
| 256 | **LDS 不足** | (256+128)×256×2 = 192KB > 160KB |

## 结论
**无改进，保持 BLOCK_K=128。** 直接进入 Round 4。
